# Review 문서 작성 가이드

## 📍 디렉토리 목적

검토 문서는 완료된 작업의 결과를 문서화하고, 성과를 측정하며, 교훈을 기록합니다.

## 📝 문서 작성 규칙

### 파일명 형식
```
[global_number]_[descriptive_name]_review.md
예: 003_authentication_system_review.md
```

### 📌 번호 할당 필수 확인
**새 review 문서 생성 전 반드시:**
1. `.claude/context/index.md` 열기
2. "다음 번호" 확인
3. 해당 번호로 문서 생성
4. index.md에 즉시 기록

### 필수 구성 요소
1. **구현 결과 요약**
2. **변경된 파일 목록**
3. **성능/품질 메트릭**
4. **발견된 문제와 해결**
5. **캡처된 지식**
6. **다음 단계 제안**

## 📊 메트릭 가이드

### 측정 항목
- **성능**: 응답시간, 처리량, 리소스 사용
- **품질**: 버그 수, 테스트 커버리지, 코드 복잡도
- **생산성**: 예상 vs 실제 시간, 재작업 횟수

### 정량화 원칙
- 가능한 모든 것을 숫자로
- 이전/이후 비교
- 목표 대비 달성률

## ✅ 체크리스트

### 작성 전
- [ ] 모든 작업 완료
- [ ] 테스트 통과
- [ ] 문서 동기화

### 작성 중
- [ ] 모든 변경사항 기록
- [ ] 메트릭 측정
- [ ] 교훈 도출

### 작성 후
- [ ] Global numbering 할당
- [ ] context/index.md 업데이트
- [ ] current.md 완료 표시
- [ ] 사용자 검토 요청

## 📌 주의사항

- 객관적 사실 위주 기록
- 실패도 정직하게 문서화
- 재현 가능한 정보 제공
- 다음 작업자를 위한 맥락 제공

## 🕐 Time MCP 필수 사용

**⚠️ Review 문서의 모든 timestamp는 `mcp__time__get_current_time` 도구로 조회하세요.**

### 적용 위치
- 작업 완료 일시
- 총 소요 시간 계산
- 성능 측정 시점

### ❌ 금지
- 시간 추측 금지
- 이전 시간 재사용 금지
