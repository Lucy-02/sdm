# Todo 문서 작성 가이드

## 📍 디렉토리 목적

할 일 문서는 승인된 계획을 실행 가능한 작업 단위로 분해합니다. 각 단계별 구체적인 구현 방법을 제시합니다.

## 📝 문서 작성 규칙

### 파일명 형식
```
[global_number]_[descriptive_name]_todo.md
예: 002_authentication_system_todo.md
```

### 📌 번호 할당 필수 확인
**새 todo 문서 생성 전 반드시:**
1. `.claude/context/index.md` 열기
2. "다음 번호" 확인
3. 해당 번호로 문서 생성
4. index.md에 즉시 기록

### 필수 구성 요소
1. **관련 계획 문서 참조**
2. **단계별 작업 분해**
3. **각 단계별 2-3개 구현 옵션**
4. **시간 추정**
5. **의존성 및 차단 요소**
6. **진행 상태 추적**

## 📄 Todo 문서 템플릿

```markdown
# [번호]_[작업명] 할 일

## 참조
- 계획: [관련 plan 문서 링크]
- 관련 문서: [DKB, structure 등]

## 작업 분해

### 단계 1: [단계명]

**목표**: [이 단계에서 달성할 것]

#### 구현 옵션 A: [접근 방법]
- [ ] 세부 작업 1
- [ ] 세부 작업 2
- [ ] 세부 작업 3

**장점**: [간단한 설명]
**단점**: [간단한 설명]

#### 구현 옵션 B: [대안 접근]
- [ ] 대체 작업 1
- [ ] 대체 작업 2

**장점**: [간단한 설명]
**단점**: [간단한 설명]

**권장**: 옵션 [A/B] - [이유]
**예상 시간**: [X 시간]
**의존성**: [필요한 선행 작업]

#### 단계 1 선택
- [ ] 옵션 A
- [ ] 옵션 B

### 단계 2: [단계명]
[동일한 구조 반복]

## 구현 순서
1. [우선순위 1]
2. [우선순위 2]
3. [우선순위 3]

## 차단 요소
- [ ] [외부 의존성]
- [ ] [필요한 권한/설정]
- [ ] [대기 중인 결정]

## 진행 상태

### 완료된 작업
- [x] [완료 작업] - [날짜시간]

### 진행 중
- [ ] [현재 작업] - 시작: [날짜시간]

### 대기 중
- [ ] [차단된 작업] - 사유: [설명]

## 테스트 계획
- [ ] 단위 테스트
- [ ] 통합 테스트
- [ ] E2E 테스트

## 완료 기준
- [ ] 모든 작업 항목 완료
- [ ] 테스트 통과
- [ ] 문서 업데이트
- [ ] 코드 리뷰 (필요시)
```

## ⚡ 진행 상태 관리

### 상태 표시
- `[ ]` : 대기 중
- `[~]` : 진행 중
- `[x]` : 완료
- `[-]` : 취소/건너뜀

### 업데이트 시점
- 작업 시작 시
- 하위 작업 완료 시
- 차단 발생 시
- 작업 완료 시

## ✅ 체크리스트

### 작성 전
- [ ] 관련 계획 문서 승인됨
- [ ] 전체 범위 이해
- [ ] 기술 스택 확정

### 작성 중
- [ ] 모든 계획 내용 포함
- [ ] 실행 가능한 단위로 분해
- [ ] 각 단계 독립적 실행 가능
- [ ] 시간 추정 현실적

### 작성 후
- [ ] Global numbering 할당
- [ ] context/index.md 업데이트
- [ ] current.md에 참조 추가
- [ ] 사용자 승인 받기

## 📌 주의사항

- 너무 세분화하지 않기 (적절한 크기 유지)
- 병렬 실행 가능한 작업 표시
- 의존성 명확히 표현
- 진행 상태 실시간 업데이트

## 🕐 Time MCP 필수 사용

**⚠️ Todo 문서의 모든 timestamp는 `mcp__time__get_current_time` 도구로 조회하세요.**

### 적용 위치
- 작업 완료 시간: `[x] 작업명 - 2025-11-28 14:35`
- 작업 시작 시간: `시작: 2025-11-28 14:00`
- 차단 발생 시간

### ❌ 금지
- 시간 추측 금지
- 이전 시간 재사용 금지
