# Plan 문서 작성 가이드

## 📍 디렉토리 목적

계획 문서는 모든 작업의 시작점입니다. 명확한 분석과 옵션 제시를 통해 최적의 방향을 결정합니다.

## 📝 문서 작성 규칙

### 파일명 형식
```
[global_number]_[descriptive_name]_plan.md
예: 001_authentication_system_plan.md
```

### 📌 번호 할당 필수 확인
**새 plan 문서 생성 전 반드시:**
1. `.claude/context/history/index.md` 열기
2. "다음 번호" 확인
3. 해당 번호로 문서 생성
4. index.md에 즉시 기록

### 필수 구성 요소
1. **정확히 3가지 옵션** 제시
2. **각 옵션별 장단점** (최소 3개씩)
3. **정량적 메트릭** (시간, 비용, 위험도)
4. **비교 분석표**
5. **명확한 권장사항**
6. **사용자 선택 체크박스**

## ✅ 체크리스트

### 작성 전
- [ ] 요구사항 명확히 이해
- [ ] 관련 기존 문서 확인
- [ ] 제약 조건 파악

### 작성 중
- [ ] 3가지 옵션 모두 실현 가능한가?
- [ ] 각 옵션의 장단점 3개 이상?
- [ ] 정량적 메트릭 포함?
- [ ] 비교표 작성?

### 작성 후
- [ ] Global numbering 할당
- [ ] history/index.md 업데이트
- [ ] current.md에 참조 추가
- [ ] 사용자 승인 대기

## 📌 주의사항

- 모든 옵션은 실제 구현 가능해야 함
- 극단적인 옵션 지양 (너무 간단 vs 너무 복잡)
- 객관적이고 균형잡힌 분석
- 기술적 근거 명확히 제시
